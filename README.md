# メカノAI (MECHANO-AI) - 車両整備サポート

車検証のQRコードをスマホで読み取り、その車種の「定番故障」「重点点検ポイント」「リコール確認リンク」を即表示する整備士向け現場ツール(PWA)。

## 機能

- **QR読み取り**: ライブカメラ。国交省「二次元コード項目定義」(2023.1版)準拠の正式パーサー
  - 二次元コード2 → 登録番号・車台番号
  - 二次元コード3 → 型式・初度登録年月・有効期間満了日・型式指定/類別区分番号
  - 仕様外のQRはヒューリスティック判定 + RAWチップからの手動割り当て
- **有効期限警告**: 満了日が90日以内 or 期限切れで赤バッジ表示
- **OCRフォールバック**: QRが読めない時は券面撮影 → Tesseract OCR(端末内・完全無料。初回のみモデルDLに通信が必要)
- **故障診断タブ**:
  - **ダイアグコード検索** — 診断機画面を撮影 or スクショ貼り付け(ペースト) → OCRでDTC(P/C/B/U)を自動抽出 → 内蔵DTC辞書(`db/dtc.json`、約65コード+系統フォールバック)で意味・原因候補・確認手順を表示
  - **問診解析** — お客様の症状を文字入力 → 症状辞書(`db/symptoms.json`、29症状)と照合し原因系統・切り分け手順を提示
  - 車検証スキャン済みなら、その車種の持病との照合と「型式+コード」での事例検索リンクも生成
- **メンテナンス諸元**: 車種ごとにオイル量・交換目安・ホイールナット締付トルク・クーラント量等を表示(**参考値** — 年式・エンジンで異なるため整備書での確認が前提。DB編集で自社の確定値に上書き可)
- **車種ノウハウDB**: `db/vehicles.json` に内蔵24車種(大型/中型/小型トラック・ハイエース等の商用車・プリウス等の普通車・N-BOX等の軽自動車)。アプリ内「DB編集」タブで追加/編集/削除(localStorage保存)、JSONエクスポート/インポート可
- **スキャン履歴**: 型式・車台番号・日時のみ保存(端末内)。タップで再表示、個別削除可
- **リコールリンク**(2026年6月検証済): 国交省 `renrakuda.mlit.go.jp` / 各メーカー公式
- **PWA**: ホーム画面追加でフルスクリーン起動、オフライン動作(車両DBもキャッシュ)

## デプロイ (GitHub Pages)

ビルド不要。静的ファイルのみ。

1. このフォルダをGitHubリポジトリにpush
   ```
   git init
   git add .
   git commit -m "initial"
   git remote add origin https://github.com/<あなたのユーザー名>/<リポジトリ名>.git
   git push -u origin main
   ```
2. GitHubのリポジトリ → Settings → Pages → Source を「Deploy from a branch」、Branch を `main` / `(root)` にして Save
3. 数分後 `https://<ユーザー名>.github.io/<リポジトリ名>/` で公開されます

※ ライブカメラはHTTPS必須のため、GitHub Pages(HTTPS)での公開がそのまま動作条件を満たします。

## スマホでの使い方

1. 公開URLをスマホで開く → ブラウザのメニューから「ホーム画面に追加」
2. 「📷 ライブスキャン」→ 車検証のQR(複数ある場合は順番に全部)を枠内へ → 「停止して解析」
3. QRがかすれて読めない場合は「🔍 OCRで読み取り」(券面撮影)

## データとプライバシー

- 所有者の氏名・住所(暗号化QR領域)は読取・保存しません
- 履歴・カスタムDB・APIキーはすべて端末内(localStorage)のみ
- リコールの有無は必ず公式リンク先で車台番号により確認してください

## ファイル構成

```
index.html        画面
style.css         スタイル(コーションプレート風デザイン)
app.js            ロジック(QRパーサー/OCR/DB/履歴)
db/vehicles.json  内蔵車種ノウハウDB
manifest.json     PWAマニフェスト
sw.js             Service Worker(オフラインキャッシュ)
icons/            アイコン(SVG + PNG)
```

## 車両DBの編集

アプリ内「DB編集」タブ推奨。直接 `db/vehicles.json` を編集する場合:

```json
{
  "match": "^(CY[LJZM]|CXZ)",   // 型式の車種コード部分にマッチする正規表現
  "name": "いすゞ ギガ",
  "maker": "isuzu",             // リコールリンク用 (isuzu/hino/fuso/ud/nissan/...)
  "faults": ["定番故障…"],
  "checks": ["点検ポイント…"],
  "specs": [{ "k": "エンジンオイル量", "v": "約13L（F/E交換時）" }],
  "notes": "自由記述(任意)"
}
```

`sw.js` の `CACHE` バージョン文字列を上げると全端末のキャッシュが更新されます。
